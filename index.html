Glosario

Asignación:
Acción de darle un valor a una variable.
ASCII(American Standard Code for Information Interchange):
Código que representa caracteres mediante números, utilizado para el intercambio de texto entre computadoras.
Archivo (File):
Conjunto de datos almacenados en disco, que puede ser leído o escrito por un programa.
Algoritmo recursivo:
Algoritmo que se llama a sí mismo para resolver subproblemas de la misma naturaleza.
Atributo:
Propiedad o característica de un objeto en programación orientada a objetos.
Acceso:
La posibilidad de interactuar con un recurso o sistema, ya sea para leer, escribir, o modificar información.
Abstract class:
Una clase que no puede ser instanciada directamente, sino que sirve como base para otras clases.
Asincrónico:
Un proceso que se ejecuta de forma independiente del resto del programa, sin bloquear el flujo principal.
Array asociativo:
Un tipo de array donde los elementos no se acceden por un índice numérico, sino por una clave o nombre.
Array:
Una estructura de datos que almacena una colección de elementos del mismo tipo, accedidos por un índice numérico.
Aplicación:
Un programa de software diseñado para realizar una tarea específica, como editar documentos o navegar por internet.
API:
Una interfaz que permite a diferentes aplicaciones comunicarse y compartir información.
Algoritmo:
Una secuencia de pasos que, si se siguen correctamente, conducen a la solución de un problema o la realización de una tarea específica.
Backend:
Una secuencia de pasos que, si se siguen correctamente, conducen a la solución de un problema o la realización de una tarea específica.
Bug:
Error o fallo en el código de un software que causa un comportamiento incorrecto o inesperado.
Build (Compilación):
Proceso de convertir el código fuente en un programa ejecutable. También se refiere a una versión específica del software.
Backup (Copia de seguridad):
Copia de los datos o del software realizada para prevenir la pérdida de información ante fallos o errores.
Bitácora (Log):
Registro que almacena información sobre eventos o actividades del software, útil para monitoreo y depuración.
Biblioteca (Library):
Conjunto de funciones y recursos reutilizables que pueden integrarse en un programa para ampliar sus capacidades.
Beta:
Versión preliminar de un software, distribuida para pruebas antes del lanzamiento oficial. Puede contener errores conocidos o desconocidos.
Bundle (Paquete):
Conjunto de archivos y recursos que se agrupan para su distribución o instalación como un solo producto.
Bus de datos:
Canal por donde se transfieren datos entre los componentes del sistema, como CPU, memoria y periféricos.
Bloque de código:
Conjunto de instrucciones agrupadas y delimitadas, que se ejecutan como una unidad dentro del software (por ejemplo, dentro de funciones, bucles, etc.).
Código fuente:
Conjunto de instrucciones escritas por un programador en un lenguaje de programación, que luego se compila o interpreta para crear software.
Compilador:
Programa que traduce el código fuente a un lenguaje máquina (ejecutable) para que pueda ser entendido y ejecutado por una computadora.
Clase:
Estructura en programación orientada a objetos que define atributos y comportamientos de un tipo de objeto.
Condición:
Expresión lógica que determina si se ejecuta o no un bloque de código, como en una estructura if.
Ciclo (Loop o Bucle):
Estructura que repite un bloque de instrucciones mientras se cumpla una condición (como while, for, etc.).
Constante:
Valor que no cambia durante la ejecución del programa. Se define una vez y permanece igual.
Cliente (Client):
Programa o dispositivo que solicita servicios o recursos a un servidor dentro de una red.
Callback:
Función que se pasa como argumento a otra función y se ejecuta en un momento específico, usualmente cuando ocurre un evento.
Commit:
Acción en sistemas de control de versiones (como Git) para guardar los cambios realizados al código fuente.
Constructor:
Método especial en una clase que se ejecuta automáticamente al crear un nuevo objeto, normalmente usado para inicializar atributos.
Depuración (Debugging):
Proceso para encontrar y corregir errores en un programa
Datos:
Información que una aplicación utiliza o genera.
Driver:
Software que permite al sistema operativo comunicarse con un dispositivo físico.
Directorio:
Carpeta donde se almacenan archivos o subdirectorios.
Desarrollador:
Persona que escribe y mantiene el código de un software.
Distribución:
Versión de un sistema operativo o software empaquetada para su instalación.
Documento:
Programa o dispositivo que solicita servicios o recursos a un servidor dentro de una red.
Dato binario:
Información representada en 0s y 1s.
Diseño de software:
Proceso de planificar la estructura y comportamiento de una aplicación.
Depurador (Debugger):
Herramienta usada para analizar el comportamiento de un programa.
Ejecución:
Momento en que un programa se está ejecutando.
Error de sintaxis:
Mal uso del lenguaje de programación que impide compilar o ejecutar.
Entorno de desarrollo (IDE):
Aplicación que incluye herramientas para programar.
Encriptación:
Técnica que transforma datos en una forma segura e ilegible.
Estructura de datos:
Formas de organizar y almacenar datos (listas, pilas, colas, etc.).
Evento:
Acción que desencadena un bloque de código (como un clic).
Entrada:
Información que el usuario o dispositivo proporciona al sistema.
Excepción:
Error que ocurre durante la ejecución de un programa.
Exportar:
Guardar datos o estructuras en un formato accesible por otros sistemas.
Etiqueta (Tag):
En programación web, marca que define elementos HTML.
Función:
Bloque de código reutilizable que realiza una tarea específica.
Framework:
Estructura de trabajo con herramientas predefinidas para programar más rápido.
Frontend:
Parte del software visible para el usuario final.
Formulario:
Componente visual que permite ingresar datos.
Fuente (Source):
Código fuente de un programa.
Factorización:
Técnica algorítmica usada en matemáticas y programación.
Fichero (File):
Archivo que almacena datos de manera permanente.
Filtro:
Error que ocurre durante la ejecución de un programa.
Fetch:
Método para obtener datos desde un servidor.
Float (Flotante):
Tipo de dato numérico que permite decimales.
Git:
Momento en que un programa se está ejecutando.
GitHub:
Plataforma para alojar proyectos con Git.
Gestor de base de datos:
Software que administra el almacenamiento y acceso a datos.
GUI (Interfaz gráfica de usuario):
Interfaz visual para interactuar con un sistema.
Gradient (Gradiente):
Transición progresiva entre colores, usado en diseño de interfaces.
Gateway:
Acción que desencadena un bloque de código (como un clic).
Global (Variable global):
Variable accesible desde cualquier parte del código.
Generador (Generator):
Error que ocurre durante la ejecución de un programa.
Graficador:
Herramienta que crea visualizaciones a partir de datos.
Gantt (Diagrama de Gantt):
Herramienta de planificación usada en gestión de proyectos.
Hardware:
Componentes físicos de un sistema informático.
HTML:
Lenguaje de marcado para construir páginas web.
Herencia:
Mecanismo en POO donde una clase hereda características de otra.
Hash:
Función que convierte datos en una cadena de longitud fija.
Heap:
Estructura de datos o zona de memoria dinámica.
Header (Encabezado):
Información previa al contenido principal (en archivos o protocolos).
Host:
Equipo que ofrece recursos o servicios en una red.
Hotfix:
Corrección rápida de un error crítico en software.
HTTP:
Protocolo para la transmisión de datos en la web.
Hybrid app:
Aplicación móvil construida con tecnologías web y empaquetada como app nativa.
IDE (Entorno de desarrollo):
Aplicación que reúne herramientas para programar.
Intérprete:
Software que ejecuta código directamente sin compilarlo.
Iteración:
Repetición de un bloque de instrucciones.
Interfaz:
Punto de contacto entre sistemas o entre el software y el usuario.
Índice:
Posición numérica en una estructura como un array.
Importar:
Usar librerías o módulos externos en un programa.

Objeto creado a partir de una clase.
Inyección SQL:
Técnica de ataque que manipula consultas a bases de datos.
Inicialización:
Asignar un valor inicial a una variable o estructura.
Inteligencia Artificial:
Rama de la informática que simula la inteligencia humana.

Lenguaje de programación orientado a objetos multiplataforma.

Kernel:
Núcleo del sistema operativo que gestiona recursos.

Programa malicioso que graba las teclas pulsadas.
Key (Clave):

Keyword (Palabra clave):

Kotlin:

Lenguaje de programación:

Licencia de software:

Login:
